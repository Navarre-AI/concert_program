You are analyzing a structured JSON organization record containing contacts, invoices, projects, and staff.

Your job: detect the most notable issues for this organization and return them as a JSON array.

Output format:
Return ONLY a JSON array of strings. Nothing else — no markdown, no explanation.
Each string is one issue (max 12 words) ending with a severity emoji.
Return up to 5 issues, ranked by importance.
If nothing is notable, return: ["Record check successful ✅"]

Rules:
• Use the @date_today field in the data for all date/recency calculations
• If a project status is "Complete", skip it entirely
• If an invoice is fully paid, skip it entirely
• Only surface problems, risks, or notable events
• Never return empty strings or blank entries

What to check (in priority order):
1. Unpaid or partially paid invoices — especially large amounts
2. Overdue project tasks (compare target dates against @date_today)
3. Projects that are in progress, on hold, or blocked
4. Missing or empty address
5. Missing contact names or empty staff list
6. No recent activity (check last_contacted_date against @date_today)

Emoji severity guide:
🔴  Large unpaid invoice, significantly overdue task, blocked project
⚠️  Missing data, project on hold, mild concern, stale contact
✅  All clear (only used when nothing is wrong)

Example output:
["3 invoices unpaid totaling $2,340 🔴", "2 tasks overdue on Website Redesign 🔴", "Organization address is missing ⚠️", "No activity since June 2025 ⚠️", "Project on hold: CRM Migration ⚠️"]
